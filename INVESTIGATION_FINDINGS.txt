REDUNDANCY 2: RSI Calculation
──────────────────────────────
Agent claim: "16× redundancy in features"
Actual measurement: 88 redundant calls (8 windows × 11 TFs)
Status: ❌ NOT OPTIMIZED (HIGH PRIORITY)

Problem:
  - Computed once at top level: 0.5ms
  - Recomputed 88 more times in feature extraction: ~0.05-0.1ms each
  - Direct cost: 4-9ms
  - Indirect cost: RSI is used for divergence and bounce detection
  - Coupled indirect cost: ~10-15% of per-window time (20-40ms)
  - Total realistic cost: 40-80ms per position

Root cause: extract_tsla_channel_features() recomputes RSI series internally

Location: v7/features/full_features.py, line 258
```python
def extract_tsla_channel_features(tsla_df, timeframe, window, ...):
    rsi_series = calculate_rsi_series(tsla_df['close'].values, period=14)  # REDUNDANT!
    divergence = detect_rsi_divergence(tsla_df['close'].values, rsi_series)
    bounces = detect_bounces_with_rsi(tsla_df, channel, rsi_series)
```

Potential fix: Add parameter to accept pre-computed RSI series
Estimated savings: 40-80ms (3-6% improvement)
Complexity: Low (one parameter addition)
Risk: Low (backward compatible)

ACTUAL BOTTLENECK IDENTIFIED
================================================================================

The main performance opportunity is NOT among the "three redundancies" but
rather the per-TF feature extraction that happens for each window.

Each window iteration does this 11 times (once per timeframe):
  1. Detect channel with specific window size: ~1ms
  2. Calculate RSI series: 0.5ms [REDUNDANT - same data, same result]
  3. Detect RSI divergence: ~0.05ms [DEPENDS ON RSI - REDUNDANT]
  4. Detect bounces: ~15ms [DEPENDS ON RSI - REDUNDANT]
  5. Check containment: ~2-3ms
  6. Track exits: ~10-15ms
  7. Calculate break triggers: ~1-2ms

Per-window overhead: ~30-40ms (40% of window time is RSI-dependent)
