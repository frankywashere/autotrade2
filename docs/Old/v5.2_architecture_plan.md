### What's Still Missing (Transition Labels Only)

The model needs to learn what happens AFTER a channel breaks:

| Label | Purpose | Values |
|-------|---------|--------|
| `transition_type` | What kind of transition? | 0=continue, 1=switch_tf, 2=reverse, 3=sideways |
| `switch_to_tf` | If switching, which TF? | '5min', '15min', ..., '3month', or None |
| `new_direction` | Phase 2 direction | 0=bull, 1=bear, 2=sideways |
| `new_slope` | Phase 2 slope magnitude | float |

### What to Keep vs Generate

| Label Type | Status | Action |
|------------|--------|--------|
| Duration labels | ✅ Already exists | USE existing `duration_bars` from hierarchical labels |
| Quality metrics | ✅ Already exists | USE existing `channel_r_squared`, `channel_valid`, etc. |
| Transition labels | ❌ Missing | GENERATE new transition labels |
| Phase 2 labels | ❌ Missing | GENERATE as part of transition labels |

### Transition Label Generation (New)

Since duration labels exist, we only need to add transition classification.

**Analogy: The Relay Race**

Duration labels tell you "Runner A ran for 15 seconds before stopping."
Transition labels tell you "Then Runner B took the baton" or "Runner A turned around."

**New Function**: Add to `src/ml/features.py`

```python
def generate_transition_labels(
    self,
    continuation_labels_dir: Path,
    output_dir: Path = None
) -> Dict[str, Path]:
    """
    Generate transition labels for v5.2 Multi-Phase Compositor (NEW).

    USES existing hierarchical continuation labels (duration_bars, channel_slope).
    ADDS transition classification: what happens AFTER each channel breaks?

    Args:
        continuation_labels_dir: Directory with existing continuation_labels_{tf}_*.pkl files
        output_dir: Where to save transition labels

    Returns:
        Dict mapping timeframe -> transition labels file path
    """
    TRANSITION_CONTINUE = 0   # Same channel extends (duration reached max)
    TRANSITION_SWITCH_TF = 1  # Different TF's channel takes over
    TRANSITION_REVERSE = 2    # Same TF, opposite direction (bull→bear or vice versa)
    TRANSITION_SIDEWAYS = 3   # Same TF, enters consolidation

    SLOPE_THRESHOLD = 0.0005  # Below this = sideways

    def slope_to_direction(slope):
        if slope > SLOPE_THRESHOLD: return 0  # BULL
        elif slope < -SLOPE_THRESHOLD: return 1  # BEAR
        else: return 2  # SIDEWAYS

    saved_files = {}

    for tf in HIERARCHICAL_TIMEFRAMES:
        # Load existing continuation labels for this TF
        cont_files = list(continuation_labels_dir.glob(f"continuation_labels_{tf}_*.pkl"))
        if not cont_files:
            continue

        cont_labels = pd.read_pickle(cont_files[0])
        transition_labels = []

        for i in range(len(cont_labels) - 10):
            row = cont_labels.iloc[i]
            duration = int(row['duration_bars'])
            current_slope = row['channel_slope']
            current_direction = slope_to_direction(current_slope)

            # Look at what happened after the break
            if i + duration + 5 < len(cont_labels):
                future_row = cont_labels.iloc[i + duration + 5]
                future_slope = future_row['channel_slope']
                future_direction = slope_to_direction(future_slope)

                # Classify transition
                if current_direction != future_direction:
                    if future_direction == 2:  # SIDEWAYS
                        transition_type = TRANSITION_SIDEWAYS
                    else:
                        transition_type = TRANSITION_REVERSE
                else:
                    transition_type = TRANSITION_CONTINUE

                # Check if different TF has higher quality
                # (Would need to load other TF's labels for full implementation)
                switch_to_tf = None  # Simplified for now

                transition_labels.append({
                    'timestamp': row.name,
                    'duration_bars': duration,
                    'transition_type': transition_type,
                    'switch_to_tf': switch_to_tf,
                    'current_direction': current_direction,
                    'new_direction': future_direction,
                    'new_slope': future_slope,
                })

        # Save
        labels_df = pd.DataFrame(transition_labels)
        output_path = output_dir / f"transition_labels_{tf}.pkl"
        labels_df.to_pickle(output_path)
        saved_files[tf] = output_path

        print(f"  {tf}: {len(labels_df):,} transition labels")

    return saved_files
```

**VIX Refresh Strategy**:

| Mode | Refresh Rate | Data Source |
|------|--------------|-------------|
| Training | Once per epoch | Historical CSV (daily) |
| Live Inference | Every hour | yfinance ^VIX (intraday) |

```python
# Live inference VIX refresh
def refresh_vix_hourly(self):
    """Update VIX sequence with latest intraday value."""
    import yfinance as yf

    # Get today's latest VIX
    vix_today = yf.download("^VIX", period="1d", interval="1h", progress=False)
    if len(vix_today) > 0:
        latest_vix = vix_today['Close'].iloc[-1]

        # Update last day in sequence
        self.vix_sequence[-1, 0] = latest_vix  # vix_close column

        # Re-run VIX CfC
        with torch.no_grad():
            self.hidden_vix = self.vix_layer(self.vix_sequence.unsqueeze(0))[0][:, -1, :]
```

**Analogy**: The seismograph that tracks market tremors. Historical training uses the full daily record. Live inference updates the latest reading every hour.

```python
# live_events.py

from datetime import datetime, date, timedelta
from typing import List, Dict, Optional
import requests

from .fomc_calendar import get_next_fomc_meeting, get_all_fomc_dates_2025


class LiveEventFetcher:
    """
    Fetch upcoming market events for live inference.

    Analogy: Train schedule showing known disruptions ahead.
    """

    # Event type IDs (for embedding lookup)
    EVENT_TYPES = {
        'fomc': 0,
        'earnings': 1,
        'delivery': 2,
        'cpi': 3,
        'nfp': 4,
        'other': 5,
    }

    def __init__(self, finnhub_api_key: Optional[str] = None):
        self.finnhub_key = finnhub_api_key
        self.fomc_dates = get_all_fomc_dates_2025()

        # TSLA delivery dates (predictable quarterly: early Jan/Apr/Jul/Oct)
        self.delivery_dates = [
            date(2025, 1, 2), date(2025, 4, 2),
            date(2025, 7, 2), date(2025, 10, 2),
            date(2026, 1, 2),
        ]

    def fetch_upcoming_events(
        self,
        ticker: str = 'TSLA',
        days_ahead: int = 30
    ) -> List[Dict]:
        """
        Fetch all upcoming events within days_ahead.

        Returns:
            List of event dicts with keys:
                - type: Event type string
                - type_id: Integer for embedding lookup
                - date: ISO date string
                - days_until: Days from today
        """
        events = []
        today = date.today()

        # 1. FOMC meetings (from scraper)
        next_fomc = get_next_fomc_meeting()
        if 'days_until' in next_fomc and next_fomc['days_until'] <= days_ahead:
            events.append({
                'type': 'fomc',
                'type_id': self.EVENT_TYPES['fomc'],
                'date': next_fomc['next_meeting'],
                'days_until': next_fomc['days_until'],
            })

        # 2. TSLA earnings (Finnhub API if available)
        if self.finnhub_key:
            earnings = self._fetch_earnings(ticker, days_ahead)
            if earnings:
                events.append(earnings)

        # 3. TSLA deliveries (predictable quarterly)
        for delivery_date in self.delivery_dates:
            days = (delivery_date - today).days
            if 0 <= days <= days_ahead:
                events.append({
                    'type': 'delivery',
                    'type_id': self.EVENT_TYPES['delivery'],
                    'date': delivery_date.isoformat(),
                    'days_until': days,
                })
                break  # Only next delivery

        # Sort by proximity
        events.sort(key=lambda x: x['days_until'])

        return events

    def _fetch_earnings(self, ticker: str, days_ahead: int) -> Optional[Dict]:
        """Fetch next earnings date from Finnhub."""
        try:
            url = "https://finnhub.io/api/v1/calendar/earnings"
            params = {
                'symbol': ticker,
                'from': date.today().isoformat(),
                'to': (date.today() + timedelta(days=days_ahead)).isoformat(),
                'token': self.finnhub_key,
            }

            response = requests.get(url, params=params, timeout=10)
            data = response.json()

            if 'earningsCalendar' in data and len(data['earningsCalendar']) > 0:
                next_earnings = data['earningsCalendar'][0]['date']
                earnings_date = datetime.strptime(next_earnings, '%Y-%m-%d').date()
                days = (earnings_date - date.today()).days

                return {
                    'type': 'earnings',
                    'type_id': self.EVENT_TYPES['earnings'],
                    'date': next_earnings,
                    'days_until': days,
                }
        except Exception as e:
            print(f"Warning: Finnhub earnings fetch failed: {e}")

        return None
```

**Event Embedding Network**:

```python
# In hierarchical_model.py

class EventEmbedding(nn.Module):
    """
    Convert upcoming events to learned embedding.

    Analogy: The looming shadow - events cast longer shadows as they approach.
    """

    def __init__(self, event_types: int = 6, embed_dim: int = 32):
        super().__init__()

        # Event type embedding
        self.type_embed = nn.Embedding(event_types, 16)

        # Timing features encoder
        # Input: [days_until_normalized, urgency, decay]
        self.timing_net = nn.Sequential(
            nn.Linear(3, 16),
            nn.ReLU(),
            nn.Linear(16, 16),
        )

        # Fusion
        self.fusion = nn.Sequential(
            nn.Linear(32, embed_dim),
            nn.ReLU(),
        )

    def forward(self, events: List[Dict], batch_size: int, device) -> torch.Tensor:
        """
        Encode events into embedding.

        Args:
            events: List of event dicts from LiveEventFetcher
            batch_size: Current batch size
            device: Target device

        Returns:
            [batch_size, 32] embedding
        """
        if not events or len(events) == 0:
            return torch.zeros(batch_size, 32, device=device)

        # Encode top 3 nearest events
        top_events = sorted(events, key=lambda x: x['days_until'])[:3]

        embeddings = []
        for event in top_events:
            # Type embedding
            type_id = torch.tensor([event['type_id']], device=device)
            type_emb = self.type_embed(type_id)  # [1, 16]

            # Timing features
            days = event['days_until']
            timing = torch.tensor([[
                days / 30.0,                        # Normalized days
                1.0 / (abs(days) + 1),              # Urgency (closer = higher)
                float(np.exp(-abs(days) / 7.0)),   # Decay (7-day half-life)
            ]], device=device)
            timing_emb = self.timing_net(timing)  # [1, 16]

            # Combine
            event_emb = torch.cat([type_emb, timing_emb], dim=-1)  # [1, 32]
            embeddings.append(event_emb)

        # Average all event embeddings
        combined = torch.stack(embeddings).mean(dim=0)  # [1, 32]

        # Expand to batch size
        return self.fusion(combined).expand(batch_size, -1)
```

### Component 5: Validity Head (Forward-Looking, NOT Circular)

**Clarification**: Quality score and validity answer DIFFERENT questions:

| Metric | Question | Direction |
|--------|----------|-----------|
| `quality_score` | "How well did channel fit PAST data?" | Backward-looking |
| `validity` | "Will this channel hold going FORWARD?" | Forward-looking |

**Example where they differ:**
```
quality_score = 0.95 (excellent historical fit)
BUT: Earnings tomorrow, VIX spiked, price at boundary
→ validity should be LOW (~0.3)
```

**This is NOT circular** because validity uses quality as ONE input among many:

```python
# Validity head learns: "High quality + high VIX + near event = low validity"
self.validity_heads = nn.ModuleDict()
for tf in self.TIMEFRAMES:
    self.validity_heads[tf] = nn.Sequential(
        nn.Linear(hidden_size + 128 + 32 + 2, 64),  # hidden + VIX + events + [quality, position]
        nn.ReLU(),
        nn.Dropout(0.1),
        nn.Linear(64, 32),
        nn.ReLU(),
        nn.Linear(32, 1),
        nn.Sigmoid()  # 0-1: Will this channel hold?
    )

# Forward:
validity_input = torch.cat([
    hidden,              # CfC learned patterns
    hidden_vix,          # VIX regime
    event_embed,         # Upcoming catalysts
    quality_score,       # Historical fit (one input, not the answer!)
    position,            # Where in channel (0=lower, 1=upper)
], dim=-1)

validity = self.validity_heads[tf](validity_input)

# Supervision: Did channel actually hold for duration_bars?
# target_validity = 1.0 if channel held, 0.0 if broke early
```

**What the validity head learns:**
- "High quality + low VIX + no events + mid-channel position → HIGH validity"
- "High quality + high VIX + earnings tomorrow + at boundary → LOW validity"
- "Low quality + any context → LOW validity"

### Component 6: Probabilistic Duration

**Change**: Duration head outputs mean + uncertainty, not point estimate.

**Analogy**: Ferry schedule - "Probably 15 minutes, but could be 10-20."

```python
# Duration heads (per TF)
self.duration_heads = nn.ModuleDict()
for tf in self.TIMEFRAMES:
    self.duration_heads[tf] = nn.ModuleDict({
        'mean': nn.Sequential(
            nn.Linear(hidden_size + 128 + 32, 64),  # hidden + VIX + events
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(64, 1),
            nn.Softplus(),  # Positive duration
        ),
        'log_std': nn.Sequential(
            nn.Linear(hidden_size + 128 + 32, 64),
            nn.ReLU(),
            nn.Linear(64, 1),
        ),
    })

# Forward:
context = torch.cat([hidden, hidden_vix, event_embed], dim=-1)
duration_mean = self.duration_heads[tf]['mean'](context)
duration_log_std = self.duration_heads[tf]['log_std'](context)
duration_std = torch.exp(duration_log_std).clamp(1, 20)

# Three projection scenarios
conservative = (duration_mean - duration_std).clamp(1, 48)
expected = duration_mean.clamp(1, 48)
aggressive = (duration_mean + duration_std).clamp(1, 48)

# Duration confidence = inverse of relative uncertainty
duration_confidence = 1.0 - (duration_std / duration_mean).clamp(0, 0.95)
```

**Loss Function (Gaussian NLL)**:

```python
# Target: actual duration from labels
target_duration = batch['duration_labels'][tf]

# Gaussian negative log-likelihood
duration_nll = 0.5 * ((target_duration - duration_mean) / duration_std) ** 2 + duration_log_std
loss_duration = duration_nll.mean()
```

### Component 7: Multi-Phase Compositor

**Purpose**: Predict what happens when current channel ends.

**Channel Directions**:
- BULL (slope > 0.0005): Upward trending
- BEAR (slope < -0.0005): Downward trending
- SIDEWAYS: Consolidation

**Transition Types**:
- CONTINUE (0): Same channel extends
- SWITCH_TF (1): Different timeframe's channel takes over
- REVERSE (2): Same TF, opposite direction (bull→bear or bear→bull)
- SIDEWAYS (3): Same TF, enters consolidation

```python
class MultiPhaseCompositor(nn.Module):
    """
    Predict channel transitions and Phase 2 projections.

    Analogy: Relay race handoff detector - who takes the baton next?
    """

    def __init__(self, hidden_size: int, n_timeframes: int = 11):
        super().__init__()

        # Transition type predictor
        # Input: all TF hiddens + VIX + events
        input_dim = hidden_size * n_timeframes + 128 + 32
        self.transition_head = nn.Sequential(
            nn.Linear(input_dim, 256),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(256, 64),
            nn.ReLU(),
            nn.Linear(64, 4),  # [continue, switch_tf, reverse, sideways]
        )

        # TF switch predictor (which TF to switch to)
        self.tf_switch_head = nn.Sequential(
            nn.Linear(hidden_size * n_timeframes, 128),
            nn.ReLU(),
            nn.Linear(128, n_timeframes),  # Prob per TF
        )

        # Phase 2 direction predictor
        self.direction_head = nn.Sequential(
            nn.Linear(hidden_size + 128, 64),  # TF hidden + VIX
            nn.ReLU(),
            nn.Linear(64, 3),  # [bull, bear, sideways]
        )

        # Phase 2 slope magnitude
        self.phase2_slope_head = nn.Sequential(
            nn.Linear(hidden_size + 128, 32),
            nn.ReLU(),
            nn.Linear(32, 1),
        )

    def forward(
        self,
        all_hidden: Dict[str, torch.Tensor],
        hidden_vix: torch.Tensor,
        event_embed: torch.Tensor,
        current_tf_idx: int
    ) -> Dict[str, torch.Tensor]:
        """
        Predict transition type and Phase 2 parameters.
        """
        # Stack all hidden states
        h_all = torch.cat([all_hidden[tf] for tf in TIMEFRAMES], dim=-1)
        context = torch.cat([h_all, hidden_vix, event_embed], dim=-1)

        # Transition type
        transition_logits = self.transition_head(context)
        transition_probs = F.softmax(transition_logits, dim=-1)

        # TF switch probabilities
        tf_switch_logits = self.tf_switch_head(h_all)
        tf_switch_probs = F.softmax(tf_switch_logits, dim=-1)

        # Phase 2 direction
        current_hidden = all_hidden[TIMEFRAMES[current_tf_idx]]
        dir_context = torch.cat([current_hidden, hidden_vix], dim=-1)
        direction_logits = self.direction_head(dir_context)
        direction_probs = F.softmax(direction_logits, dim=-1)

        # Phase 2 slope
        phase2_slope = self.phase2_slope_head(dir_context)

        return {
            'transition_probs': transition_probs,      # [batch, 4]
            'tf_switch_probs': tf_switch_probs,        # [batch, 11]
            'direction_probs': direction_probs,        # [batch, 3]
            'phase2_slope': phase2_slope,              # [batch, 1]
        }
```

## Training Updates

### Loss Function (Complete)

```python
def compute_loss(predictions, output_dict, targets, loss_weights):
    """
    Complete v5.2 loss function.
    """
    # =========================================================================
    # 1. PRIMARY PREDICTION LOSS
    # =========================================================================
    loss_high = F.mse_loss(predictions[:, 0], targets['high'])
    loss_low = F.mse_loss(predictions[:, 1], targets['low'])

    primary_loss = loss_high + loss_low

    # =========================================================================
    # 2. DURATION LOSS (Gaussian NLL)
    # =========================================================================
    duration_loss = 0.0
    for tf in TIMEFRAMES:
        if f'duration_{tf}' in targets:
            mean = output_dict['duration'][tf]['mean']
            std = output_dict['duration'][tf]['std']
            target = targets[f'duration_{tf}']

            # Gaussian NLL: 0.5 * ((target - mean) / std)^2 + log(std)
            nll = 0.5 * ((target - mean) / std) ** 2 + torch.log(std)
            duration_loss += nll.mean()

    duration_loss /= len(TIMEFRAMES)  # Average across TFs

    # =========================================================================
    # 3. VALIDITY LOSS (Binary Cross-Entropy)
    # =========================================================================
    # Target: Did channel actually hold for predicted duration?
    # 1.0 if channel held, 0.0 if broke early
    validity_loss = 0.0
    for tf in TIMEFRAMES:
        if f'validity_{tf}' in output_dict and f'channel_held_{tf}' in targets:
            pred_validity = output_dict[f'validity_{tf}']
            target_held = targets[f'channel_held_{tf}'].float()  # Binary: held or not
            validity_loss += F.binary_cross_entropy(pred_validity, target_held)

    validity_loss /= len(TIMEFRAMES)

    # =========================================================================
    # 4. TRANSITION LOSS (Cross-Entropy)
    # =========================================================================
    if 'transition_type' in targets:
        transition_logits = output_dict['compositor']['transition_probs']
        transition_loss = F.cross_entropy(
            transition_logits,
            targets['transition_type'].long()
        )
    else:
        transition_loss = 0.0

    # =========================================================================
    # 4. TF SWITCH LOSS (Cross-Entropy, masked)
    # =========================================================================
    if 'switch_to_tf' in targets:
        # Only compute for samples where transition = SWITCH_TF
        switch_mask = (targets['transition_type'] == 1).float()

        if switch_mask.sum() > 0:
            tf_switch_logits = output_dict['compositor']['tf_switch_probs']
            switch_loss = F.cross_entropy(
                tf_switch_logits,
                targets['switch_to_tf'].long(),
                reduction='none'
            )
            tf_switch_loss = (switch_loss * switch_mask).sum() / (switch_mask.sum() + 1e-6)
        else:
            tf_switch_loss = 0.0
    else:
        tf_switch_loss = 0.0

    # =========================================================================
    # 5. DIRECTION LOSS (Cross-Entropy)
    # =========================================================================
    if 'new_direction' in targets:
        direction_logits = output_dict['compositor']['direction_probs']
        direction_loss = F.cross_entropy(
            direction_logits,
            targets['new_direction'].long()
        )
    else:
        direction_loss = 0.0

    # =========================================================================
    # 6. PHASE 2 SLOPE LOSS (MSE)
    # =========================================================================
    if 'new_slope' in targets:
        phase2_slope = output_dict['compositor']['phase2_slope']
        slope_loss = F.mse_loss(phase2_slope.squeeze(), targets['new_slope'])
    else:
        slope_loss = 0.0

    # =========================================================================
    # 7. ADJUSTMENT REGULARIZATION (Encourage small adjustments)
    # =========================================================================
    adjustment_reg = 0.0
    for tf in TIMEFRAMES:
        if tf in output_dict['projections']:
            adj_h = output_dict['projections'][tf]['adjustment_high']
            adj_l = output_dict['projections'][tf]['adjustment_low']
            adjustment_reg += (adj_h ** 2 + adj_l ** 2).mean()

    adjustment_reg /= len(TIMEFRAMES)

    # =========================================================================
    # TOTAL LOSS
    # =========================================================================
    total_loss = (
        loss_weights['primary'] * primary_loss +
        loss_weights['duration'] * duration_loss +
        loss_weights['validity'] * validity_loss +
        loss_weights['transition'] * transition_loss +
        loss_weights['tf_switch'] * tf_switch_loss +
        loss_weights['direction'] * direction_loss +
        loss_weights['slope'] * slope_loss +
        loss_weights['adjustment_reg'] * adjustment_reg
    )

    return total_loss, {
        'primary': primary_loss.item(),
        'duration': duration_loss.item() if isinstance(duration_loss, torch.Tensor) else duration_loss,
        'validity': validity_loss.item() if isinstance(validity_loss, torch.Tensor) else validity_loss,
        'transition': transition_loss.item() if isinstance(transition_loss, torch.Tensor) else transition_loss,
        'adjustment_reg': adjustment_reg.item(),
    }
```
