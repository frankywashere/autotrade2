## FEATURE 1: Meta-CFC (Cross-Timeframe Pattern Learner)

### Concept

**Current v5.3**: 11 CFCs + simple feedforward Compositor
**v5.4**: 11 CFCs + Meta-CFC that processes all 11 hiddens as a sequence

**Analogy**: The Orchestra Conductor

11 CFCs = 11 musicians (each plays their instrument)
Current Compositor = Music producer (mixes the tracks)
Meta-CFC = Conductor (watches HOW they play together in real-time)

---

### Implementation

```python
# After all 11 TF CFCs run:
hidden_sequence = torch.stack([
    hidden_5min,
    hidden_15min,
    hidden_30min,
    hidden_1h,
    hidden_2h,
    hidden_3h,
    hidden_4h,
    hidden_daily,
    hidden_weekly,
    hidden_monthly,
    hidden_3month
], dim=1)  # [batch, 11, 128]

# Meta-CFC processes this as a sequence
meta_wiring = AutoNCP(512, 256)  # Larger than individual CFCs
meta_cfc = CfC(input_size=128, wiring=meta_wiring, batch_first=True)

meta_out, meta_hidden = meta_cfc(hidden_sequence)
meta_understanding = meta_out[:, -1, :]  # [batch, 256]

# Use for:
# - Enhanced compositor (sees meta patterns)
# - TF selection (uses meta understanding)
# - Duration refinement (cross-TF temporal patterns)
```

---

### What Meta-CFC Would Learn

**Patterns Across Timeframe Spectrum**:
```
"When 5min choppy + 30min forming + 1h strong + daily consolidating:"
  → This pattern historically leads to 1h channel winning

"When all fast TFs (5min-1h) bearish but slow TFs (daily-monthly) bullish:"
  → Fast TFs switch to slow TF alignment within 20 bars

"When 5min→15min→30min show increasing quality scores:"
  → Trend strengthening, extend duration predictions
```

**Temporal Dynamics**:
- How patterns cascade across timeframes
- Which TF leads vs lags
- Synchronization patterns

---

### Benefits

✅ Captures meta-patterns simple attention misses
✅ Learns timeframe interaction dynamics
✅ Could improve duration/transition predictions

### Costs

❌ +500K parameters
❌ Slower training
❌ More complex architecture

**Decision Rule**: Add if v5.3 compositor accuracy < 70%

## FEATURE 2: Improved Phase 2 Projection

### Current v5.3

Phase 2 predictions exist but aren't geometrically projected:
```
Compositor says:
  - Transition: REVERSE (70%)
  - Direction: BEAR
  - Slope: -0.08

But we don't actually PROJECT the imagined bear channel!
```

---

### v5.4 Enhancement

**Actually compute Phase 2 geometric projection**:

```python
# Phase 1: Current channel (bars 0-18)
phase1_projection = geometric_project(
    slope=current_slope,
    duration=predicted_duration,  # 18 bars
    bounds=current_bounds
)
phase1_high = +3.8%
phase1_low = +0.5%

# Phase 2: Imagined post-reversal channel (bars 18-24)
if transition_type == REVERSE:
    # Use predicted new slope
    phase2_slope = compositor_output['phase2_slope']  # -0.08

    # Starting point: where Phase 1 ended
    phase1_end_price = current_price + (current_price * 0.035)  # ~+3.5%

    # Project Phase 2 channel
    phase2_projection = geometric_project(
        slope=phase2_slope,
        starting_price=phase1_end_price,
        duration=6,  # Remaining bars (24-18)
        bounds=estimate_new_bounds(phase2_slope, volatility)
    )
    phase2_high = +3.5%  # Reversal starts here
    phase2_low = +2.2%   # After 6 bars of decline

# Composite (informational only - Option B)
composite_high = max(phase1_high, phase2_high) = +3.8%
composite_low = min(phase1_low, phase2_low) = +0.5%

# But final prediction ONLY uses Phase 1
final_high = phase1_high = +3.8%
final_low = phase1_low = +0.5%
```

---

### Visualization

**Dashboard would show**:
```
30min Channel Forecast (18 bars):
  Phase 1 (0-18 bars): Bull channel
    High: +3.8% at bar 12
    Low: +0.5% at bar 3

  ─── PREDICTED TRANSITION (70% reversal) ───

  Phase 2 (18-24 bars): Bear channel (INFORMATIONAL)
    Direction: Bear (75% prob)
    Slope: -0.08
    Projected: +3.5% → +2.2% (6-bar decline)
    Note: Not included in final prediction
```

**User sees**: "What might happen after the channel ends"

---

### Benefits

✅ Complete multi-phase visualization
✅ Users understand transition scenarios
✅ Validates compositor predictions (do they make geometric sense?)

### Implementation

- Geometric projection function (reusable)
- Bound estimation for Phase 2 (from volatility + parent channels)
- Dashboard visualization enhancements

## FEATURE 3: Explicit Containment Features in Duration

### Current v5.3

Duration sees parent hiddens (implicit containment):
```python
duration_input = concat(
    hidden_30min,
    hidden_1h,      # Implicitly knows 1h channel bounds
    hidden_4h,      # Implicitly knows 4h channel bounds
    hidden_vix,
    event_embed
)
```

**Parent hiddens contain channel info** but it's **mixed with everything else**.

---

### v5.4: Explicit Containment Features

**Add direct containment metrics**:
```python
# Calculate containment explicitly
containment_1h = calculate_containment(
    small=30min_projected_bounds,  # +0.38% / -0.37%
    large=1h_projected_bounds       # +0.52% / -0.25%
)
# Returns: {violation_high: 0.0, violation_low: 0.12, score: 0.96}

containment_4h = calculate_containment(
    small=30min_projected_bounds,
    large=4h_projected_bounds
)

# Enhanced duration input
duration_input = concat(
    hidden_30min,
    hidden_1h,
    hidden_4h,
    containment_features,  # NEW: [violation_1h_high, violation_1h_low, score_1h,
                           #       violation_4h_high, violation_4h_low, score_4h]
    parent_1h_validity,    # How strong is parent?
    parent_4h_validity,
    hidden_vix,
    event_embed
)
# Total: +8 explicit features
```

---

### What This Enables

Model can learn:
```
IF: violation_1h_low = 0.12 (approaching 1h support)
AND: parent_1h_validity = 0.95 (strong parent)
AND: parent_1h_rsi = 28 (oversold)
THEN: Duration = 6 bars (bounce pattern!)

vs

IF: violation_1h_low = 0.12
AND: parent_1h_validity = 0.40 (weak parent)
THEN: Duration = 20 bars (breakthrough pattern!)
```

**Explicit > Implicit** for learning these patterns.

## FEATURE 4: Intraday VIX Updates

### Current v5.3

**VIX**: Daily updates only
```
9:30am: VIX CfC processes daily sequence → hidden_vix
3:55pm: Still using SAME hidden_vix (stale for intraday TFs!)
```

---

### v5.4: Hourly VIX Refresh

**Live mode**:
```python
# Every hour:
def refresh_vix_intraday():
    # Fetch latest VIX
    vix_live = yfinance('^VIX', period='1d', interval='1h')
    latest_vix = vix_live['Close'].iloc[-1]

    # Update last day in VIX sequence
    vix_sequence[-1, 3] = latest_vix  # vix_close column

    # Re-run VIX CfC
    vix_out, _ = vix_cfc(vix_sequence)
    hidden_vix = vix_out[:, -1, :]

    # Invalidate cached features
    predictor._cache_invalidated = True
```

**Training**:
- Still uses daily VIX (intraday not available historically)
- But model learns from daily patterns
- Live mode benefits from fresher VIX state

