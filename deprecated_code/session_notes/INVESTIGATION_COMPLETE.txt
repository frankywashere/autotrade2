THE REAL BOTTLENECK - CHANNEL DETECTION
================================================================================

Called: 88 times per position (8 windows × 11 timeframes)
Time per call: 10-15ms each (parallel detection of 8 window sizes)
Total time: 880-1320ms (this dominates label generation)

Why it's expensive:
  detect_channels_multi_window() does this in parallel:
  
    ThreadPoolExecutor(max_workers=4):
      ├─ Detect at window 10: linear regression + touch detection
      ├─ Detect at window 20: linear regression + touch detection
      ├─ Detect at window 30: linear regression + touch detection
      ├─ Detect at window 40: linear regression + touch detection
      ├─ Detect at window 50: linear regression + touch detection
      ├─ Detect at window 60: linear regression + touch detection
      ├─ Detect at window 70: linear regression + touch detection
      └─ Detect at window 80: linear regression + touch detection
  
  Each operation:
    - Linear regression: O(window) regression calculations
    - Touch detection: O(window) boundary distance checks
    - Bounce counting: O(touches) alternation detection
    - Array operations and numpy overhead
  
  Total per call: 10-15ms (not negligible)

The problem:
  generate_labels_per_tf() calls detect_channels_multi_window() once per TF
  generate_labels_multi_window() calls generate_labels_per_tf() 8 times
  
  Result: 88 expensive channel detections per position
