# v5.3.1: Bidirectional Information Flow

**Version**: v5.3.1
**Status**: Planned
**Estimated Effort**: 2-3 days (architecture change + retraining)

---

## CONCEPT

**Current v5.3**: Bottom-up only (5min → 15min → ... → 3month)
- Fast TFs inform slow TFs
- "Ground-up reporting"

**Proposed v5.3.1**: Bidirectional flow
- Pass 1 (bottom-up): 5min → 3month (build local understanding)
- Pass 2 (top-down): 3month → 5min (add macro context)
- Each CfC benefits from both directions

---

## MOTIVATION

### Why Top-Down Helps Duration Prediction:

**Example**:
```
Current (bottom-up only):
  5min CfC: "I see a bear channel, quality=0.85"
  Duration NN: "Predict 8 bars"

  Missing context: Monthly is in strong bull!
  Reality: 5min bear is just a correction, lasts only 3 bars

With top-down:
  Monthly CfC: "Strong bull trend"
    ↓
  1h CfC: "Confirms bull, minor pullback"
    ↓
  5min CfC receives: monthly + 1h context
  5min CfC: "I see bear BUT parent says bull correction"
  Duration NN: "Predict 3 bars (short correction in bull context)"
```

**Benefit**: Macro context constrains micro predictions

## IMPLEMENTATION

### Architecture Changes:

**Current CfC Sizes**:
```python
Layer 0 (5min):  input = 900 + 0 (no prev)
Layer 1 (15min): input = 900 + 128 (prev)
...
Layer 10 (3month): input = 900 + 128 (prev)
```

**Bidirectional**:
```python
# Pass 1: Bottom-up (unchanged)
5min → 15min → ... → 3month

# Pass 2: Top-down
for i, tf in reversed(enumerate(TIMEFRAMES)):
    if i < len(TIMEFRAMES) - 1:
        # Concatenate NEXT (slower) TF's hidden state
        next_tf_hidden = hidden[TIMEFRAMES[i+1]]
        x_topdown = concat(hidden[tf], next_tf_hidden)

        # Refinement layer (optional: small NN to integrate top-down info)
        hidden[tf] = refinement_net[tf](x_topdown)
```

**Alternative (Simpler)**:
```python
# Just use parent hidden in predictions (already doing this in v5.3!)
# Current v5.3 duration already sees parent hiddens
# No CfC changes needed, just ensure all predictions use parent context
```

## TRAINING CONSIDERATIONS

**Requires Retraining**: YES
- Input sizes don't change (good!)
- But information flow changes
- Gradients flow differently
- Need 100 epochs to learn new patterns

**Not a Toggle**: Can't switch after training
- The learned weights embed the information flow direction
- Switching would use weights trained for wrong flow

---

## COMPARISON: Bottom-Up vs Top-Down vs Bidirectional

### Bottom-Up (Current v5.3):
**Good for**:
- Detail aggregation
- Noise filtering (fast TFs are noisy)
- Building complexity from simplicity

**Example**: "5min + 15min both choppy → 1h sees pattern emerging"

---

### Top-Down (Proposed):
**Good for**:
- Context provision
- Macro constraints on micro
- Strategic guidance

**Example**: "Monthly bullish → 5min bear must be correction"

---

### Bidirectional (Best of Both):
**Good for**:
- Comprehensive understanding
- Micro-macro alignment
- Duration prediction with full context

**Example**:
```
Bottom-up: "5min choppy, 15min forming pattern, 30min strong"
Top-down: "But daily is consolidating, monthly is neutral"
Result: "30min channel strong locally but may be cut short by daily consolidation"
```
