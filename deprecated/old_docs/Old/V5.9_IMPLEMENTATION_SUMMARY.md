### 4. **train_hierarchical.py** - Loss Functions

**Removed:**
- geo_price loss (bound accuracy - wrong approach)

**Added:**

**A) Containment Loss** (replaces geo_price):
```python
For each sample:
  1. Get predicted bounds (upper, lower)
  2. Get actual price sequence from labels
  3. Check: How many bars stayed within bounds?
  4. Loss = 1 - (contained_bars / total_bars)

Rewards:
  • Bounds that contain price ✓
  • Partial credit if breaks then recovers ✓
  • Tighter bounds (indirectly via window selection) ✓
```

**B) Hit Probability Loss**:
```python
For each sample:
  1. Model predicts: [prob_upper, prob_midline, prob_lower]
  2. Labels provide: [hit_upper, hit_midline, hit_lower] (0/1)
  3. Loss = BCE(predictions, targets)

Learns:
  • "High r² + RSI neutral → 80% hit midline, 20% hit upper"
  • "Low r² + volatile → 30% hit upper, 30% hit lower"
```

**Loss component tracking updated:**
- `geo_price` → `containment`
- Added: `hit_probability`

## WHAT THE SYSTEM NOW DOES

### Training Loop:

**For each batch:**

1. **CfC processes 1049 features** → hidden state
2. **Duration head**: Predicts duration (e.g., 30 bars)
3. **Window selector**: Picks windows (e.g., w50 60%, w70 40%)
4. **Hit probability head**: Predicts [prob_upper, prob_midline, prob_lower]
5. **Geometric calculation**: bounds = slope × duration (from features)

6. **Containment loss**: Check if bounds contained actual prices
7. **Hit probability loss**: Check if hit predictions were accurate
8. **Duration loss**: Check if duration prediction was accurate

9. **Primary head** (baseline): Also runs but lower priority
